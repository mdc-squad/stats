# MDC Player Statistics

Веб-приложение на Next.js для просмотра и анализа статистики игроков.

## Требования

- Docker Engine 24+ и Docker Compose v2
- (Опционально) Node.js 22+ и pnpm 10+ для запуска без Docker
- Доступ к API `https://api.hungryfishteam.org/gas/mdc`

## Быстрый старт в Docker

1. Собрать и запустить контейнер:

```bash
docker compose up --build -d
```

2. Открыть приложение:

`http://localhost:3001`

По умолчанию наружу пробрасывается порт `3001`, чтобы не конфликтовать с уже занятым `3000`.
Если порт `3000` свободен, можно запустить так:

```bash
APP_PORT=3000 docker compose up --build -d
```

3. Смотреть логи:

```bash
docker compose logs -f app
```

4. Остановить и удалить контейнер:

```bash
docker compose down
```

## Docker режим разработки (Hot Reload)

Запуск dev-сервиса с HMR (без пересборки контейнера при каждом изменении):

```bash
docker compose --profile dev up app-dev
```

- Приложение будет доступно на `http://localhost:${APP_DEV_PORT:-3002}`
- Изменения в коде подхватываются автоматически (`next dev`)

Остановить dev-сервис:

```bash
docker compose stop app-dev
```

## Локальный запуск без Docker

```bash
corepack enable
pnpm install --no-frozen-lockfile
pnpm dev
```

Приложение будет доступно на `http://localhost:3000`.

## Источник данных и кэш

- Фронтенд ходит в API через локальный proxy Next.js: `/api/mdc/*` (обход CORS/браузерных ограничений).
- При необходимости можно переопределить базу через `NEXT_PUBLIC_MDC_API_BASE`.
- Приложение работает только с API:
  - `https://api.hungryfishteam.org/gas/mdc/all`
  - `https://api.hungryfishteam.org/gas/mdc/pages`
  - `https://api.hungryfishteam.org/gas/mdc/events`
  - `https://api.hungryfishteam.org/gas/mdc/players`
  - `https://api.hungryfishteam.org/gas/mdc/playersevents`
  - `https://api.hungryfishteam.org/gas/mdc/clans`
  - `https://api.hungryfishteam.org/gas/mdc/dictionaries`
- Поддержан endpoint карточки игрока:
  - `https://api.hungryfishteam.org/gas/mdc/player?playerNickname=<nickname>`
- Для `playersevents` используется постраничная загрузка:
  - `pages` возвращает количество страниц
  - `playersevents?page=N` возвращает страницу (пустые `null`-строки автоматически отфильтровываются)
- На клиенте используется кэш в `localStorage` (`mdc-api-cache-v4`) с TTL 5 минут.
- Кэш накопительный: при синхронизации новые записи добавляются к уже сохранённым, чтобы не терять историю при урезанном ответе API.
- Принудительное обновление данных с обходом кэша:
  - кнопка `Обновить API (Shift+R)` в интерфейсе
  - горячая клавиша `Shift+R` (принудительный запрос к API без сброса накопленного кэша)
  - при принудительном обновлении к API добавляется `publish=true` (режим "мимо кеша" на стороне API)
- Отдельная кнопка `Сбросить кэш` очищает локальное накопление и выполняет полную синхронизацию заново.
- В интерфейсе отображается прогресс синхронизации (по страницам протокола) и ретроспектива новых событий за последние 7 дней.

## Авто-смена сезонного стиля

- Тема интерфейса (палитра, фон, подписи в шапке/карточках) выбирается автоматически по текущей дате.
- Календарная логика ориентируется на московское время (`Europe/Moscow`).
- Поддерживаемые популярные периоды по календарю РФ:
  - `20 декабря - 14 января`: новогодний/рождественский период
  - `20 - 24 февраля`: День защитника Отечества
  - `6 - 10 марта`: Международный женский день
  - `1 - 3 мая`: Праздник Весны и Труда
  - `7 - 10 мая`: День Победы
  - `10 - 13 июня`: День России
  - `2 - 5 ноября`: День народного единства
- Вне праздничных окон применяется сезонная тема:
  - Зима: декабрь-февраль
  - Весна: март-май
  - Лето: июнь-август
  - Осень: сентябрь-ноябрь
- Снежная анимация включается только для зимних/новогодних тем.

- `Dockerfile` - multi-stage сборка (зависимости, build, runtime)
- `docker-compose.yml` - запуск production-контейнера c пробросом `${APP_PORT:-3001}:3000`
